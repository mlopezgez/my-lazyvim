Introduction

Efficient navigation is the foundation of productive coding. This configuration uses Snacks.nvim pickers powered by modern fuzzy finding algorithms. Think of pickers as your command center‚Äîthey let you find anything in seconds.

Why Navigation Matters

  • Speed: Find files in milliseconds, not seconds
  • Context: Jump to definitions, references, and symbols instantly
  • Memory: No need to remember file paths or locations
  • Discovery: Explore unfamiliar codebases efficiently
üí° Philosophy

In this setup, you rarely use :e filename or manually navigate directories. Instead, you use fuzzy finders: type a few characters, get instant results, press Enter.

File Navigation

Essential File Pickers

These are your most-used commands. Memorize them first.

Keybind Description When to Use
<leader><space> Smart Find Files Your primary file finder. Searches git files first, falls back to all files.
<leader>, Buffers Switch between open buffers (files). Faster than :ls + :b.
<leader>ff Find Files Find any file in the current directory (including ignored files).
<leader>fg Find Git Files Only search files tracked by git. Faster in large repos.
<leader>fr Recent Files Access recently opened files across all projects.
<leader>fc Find Config File Jump directly to your Neovim config files.

Search & Grep

Live Grep (Search File Contents)

Find text across your entire project. This is your primary code search tool.

Keybind Description When to Use
<leader>/ Grep (search all files) Find text in any file. Your most powerful search command.
<leader>sg Grep (alias) Same as <leader>/. Use whichever feels natural.
<leader>sw Grep word under cursor Search for the word under your cursor across all files. Works in visual mode too.
<leader>sB Grep open buffers Search only in currently open files (faster, narrower scope).

Buffer-Specific Search

Keybind Description
<leader>sb Search lines in current buffer
<leader>sD Search diagnostics in current buffer

Pattern: Finding Function Usages

  1. Navigate to the function name
  2. Press <leader>sw (grep word under cursor)
  3. Results show all occurrences across your project
  4. Navigate with Ctrl+j/Ctrl+k
  5. Press Enter to jump to any result
  6. Use Ctrl+o to jump back
üí° Tip: This is faster than LSP "Find References" for simple name searches.

LSP Navigation

Code Intelligence Navigation

LSP (Language Server Protocol) provides semantic understanding of your code. These commands work with any language that has an LSP server configured.

Keybind Description Use Case
gd Go to Definition Jump to where a function/class is defined. Your most-used LSP command.
gD Go to Declaration Jump to declaration (mainly for C/C++, where declaration differs from definition).
gr Find References Find all places where this symbol is used (smarter than grep).
gI Go to Implementation Jump to implementation (useful for interfaces/abstract classes).
gy Go to Type Definition Jump to the type definition (e.g., from variable to class definition).

Symbol Navigation

Keybind Description
<leader>ss LSP symbols (current file)
<leader>sS LSP workspace symbols (all files)
‚ö°

Pattern: Exploring Unknown Codebase

  1. Open entry point file (e.g., main.py)
  2. Place cursor on a function call
  3. Press gd to jump to definition
  4. Read the function implementation
  5. Place cursor on another symbol and repeat
  6. Use Ctrl+o to jump back, Ctrl+i to jump forward
  7. Press gr on key functions to understand their usage
üí° Tip: This "follow the breadcrumbs" approach is the fastest way to understand new code.

Troubleshooting

Picker not finding files?

  • Use <leader>ff instead of <leader><space> to search all files (including ignored ones)
  • Check if you're in the correct directory (:pwd)
  • Ensure file isn't in .gitignore if using git pickers

LSP navigation not working?

  • Check if LSP is running: :LspInfo
  • Ensure file is saved (LSP needs to parse saved files)
  • Wait for LSP to finish indexing (check statusline)
  • Fallback to grep: <leader>sw works without LSP

Too many search results?

  • Type more characters in the picker to narrow results
  • Use <leader>sB to search only open buffers
  • Use LSP pickers (e.g., <leader>ss) for symbol-specific searches
