Master version control without leaving Neovim. Learn Lazygit integration, GitHub workflows, branch management, and efficient code review practices with Snacks pickers.

Introduction

This configuration integrates Lazygit, a terminal UI for git, directly into Neovim. Combined with Snacks pickers for git operations and GitHub integration, you can handle all version control tasks without switching contexts.

Why Integrated Git Matters

  • Context Preservation: Stay in your editor while managing git
  • Visual Diffs: See changes in-editor with syntax highlighting
  • Keyboard Efficiency: No mouse needed for git operations
  • Fast Commits: Stage, commit, and push in seconds
üí° Philosophy

Use Lazygit for complex operations (staging hunks, interactive rebase, conflict resolution). Use Snacks pickers for quick lookups (branch switching, log viewing, status checks).

Lazygit Integration

Opening Lazygit

Lazygit is a full-featured terminal UI that runs inside Neovim as a floating terminal.

Keybind Description
<leader>gg Open Lazygit
üéØ

Practice Exercise: First Lazygit Session

Objective: Learn the Lazygit interface.

  1. Make a small change to any file and save
  2. Press <leader>gg to open Lazygit
  3. You'll see panels: Status, Files, Branches, Commits
  4. Navigate panels with 1-5 number keys
  5. Press ? to see all keybindings
  6. Press q to quit Lazygit

‚úì Success: You can open, navigate, and close Lazygit.

Common Lazygit Operations

Essential Lazygit Keys

Key (in Lazygit) Action
space Stage/unstage file or hunk
a Stage all changes
c Commit staged changes
P Push to remote
p Pull from remote
n Create new branch
space (on branch) Checkout branch
M Merge branch
r Rebase
? Show all keybindings
‚ö°

Pattern: Quick Commit Workflow

  1. Make your code changes in Neovim
  2. Press <leader>gg to open Lazygit
  3. Press 2 to navigate to Files panel
  4. Use j/k to navigate files
  5. Press space to stage files (or a to stage all)
  6. Press c to commit
  7. Type commit message, save and close
  8. Press P to push (if needed)
  9. Press q to quit Lazygit
üí° Tip: This entire workflow takes less than 15 seconds with practice.

Git Pickers (Snacks)

Branch & Commit Navigation

Use Snacks pickers for quick git lookups without opening Lazygit.

Keybind Description When to Use
<leader>gb Git Branches Quick branch switching without Lazygit.
<leader>gl Git Log (all commits) Browse commit history for the entire project.
<leader>gL Git Log Line See commit history for the current line under cursor.
<leader>gf Git Log File See all commits that modified the current file.
<leader>gs Git Status View changed files, jump to any to see diff.
<leader>gd Git Diff (Hunks) View all changed hunks across all files.
<leader>gS Git Stash Browse and apply stashed changes.
üéØ

Practice Exercise: Git Log Exploration

Objective: Explore your project's git history.

  1. Open any file in your project
  2. Press <leader>gf to see file history
  3. Navigate commits with Ctrl+j/Ctrl+k
  4. Press Enter on a commit to see changes
  5. Close and try <leader>gl for full project log
  6. Place cursor on an interesting line
  7. Press <leader>gL to see that line's history

‚úì Success: You can trace code changes through history.

‚ö°

Pattern: Investigating a Bug's Origin

  1. Find the buggy code line
  2. Press <leader>gL (git log line)
  3. Review commits that modified this line
  4. Identify the commit that introduced the bug
  5. Press Enter to see full commit diff
  6. Read commit message for context
  7. Check other files changed in that commit
  8. Now you understand the bug's origin!
üí° Pro Tip: This is much faster than git blame in the terminal.

Advanced Workflows

Staging Partial Changes (Hunks)

Sometimes you want to commit only part of a file's changes. Use Lazygit's hunk staging:

  1. Open Lazygit: <leader>gg
  2. Navigate to Files panel: 2
  3. Select a file with changes
  4. Press Enter to see file diff
  5. Navigate to specific hunk (change block)
  6. Press space to stage just that hunk
  7. Repeat for other hunks you want
  8. Press c to commit staged hunks

Interactive Rebase

Rewrite commit history (squash, reword, reorder commits):

  1. Open Lazygit: <leader>gg
  2. Navigate to Commits panel: 4
  3. Select a commit to rebase from
  4. Press r for rebase options
  5. Choose operation: squash, fixup, reword, edit, drop
  6. Follow Lazygit's prompts

‚ö†Ô∏è Warning

Only rebase commits that haven't been pushed to a shared branch. Rewriting public history breaks others' work.

Conflict Resolution

When you encounter merge conflicts, Lazygit provides a visual conflict resolver:

  1. After a merge/rebase with conflicts, open Lazygit
  2. Files with conflicts are marked
  3. Select a conflicted file and press Enter
  4. Choose: Ours, Theirs, or Manual edit
  5. If manual, edit in Neovim
  6. Mark as resolved and continue merge/rebase

Troubleshooting

Lazygit not opening?

  • Ensure lazygit is installed: brew install lazygit (macOS) or check your package manager
  • Verify it's in PATH: which lazygit
  • Restart Neovim after installing

GitHub pickers showing no results?

  • Install GitHub CLI: brew install gh
  • Authenticate: gh auth login
  • Ensure you're in a GitHub-tracked repository
  • Check internet connection

Git Browse opening wrong URL?

  • Ensure you're on the correct branch
  • Check remote URL: git remote -v
  • Push your branch to remote if it doesn't exist there yet
